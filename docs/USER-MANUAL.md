# NetSwitcher 用户手册

NetSwitcher 帮你在「内网以太网 + 外网 Wi-Fi」双网卡机器上自动分流：你只管说"哪些网段走哪块网卡"，剩下的（默认路由、metric、Wi-Fi 重连后重下发）它来维护。

> 本手册描述当前**嵌入式架构**（GUI 进程内嵌路由引擎 + 托盘常驻）。技术细节见 `NetSwitcher-技术方案.md`，快速上手见 `README.md`。

---

## 1. 启动

双击 `netswitcher.exe` 即可。

- **首次启动**：GUI 检测到未以管理员身份运行 → 弹出引导对话框：
  > 修改路由表需要管理员权限。点击"以管理员身份重启"。
  点按钮 → UAC 弹窗 → 同意 → 自动以管理员重启 → 全功能可用。
- **之后再打开**：直接进主界面。
- **建议**：进"设置"页 → 开"开机自启"，下次登录 Windows 自动以管理员启动，免每次 UAC。

> 为什么必须管理员？`route.exe` / `netsh.exe` 修改路由表需要管理员权限，普通用户权限改不了。NetSwitcher 把所有路由操作集中在一个提权的 GUI 进程里完成。

## 2. 界面总览

窗口顶栏（可拖动移动窗口）：左侧"NetSwitcher"标题，右侧状态药丸 + 三个窗口按钮（− 最小化 / □ 最大化还原 / ✕ 缩到托盘）。

左侧导航：**状态 / 配置 / 路由表 / 诊断 / 日志 / 设置**。

## 3. 配置第一条规则

1. 进入 **配置** 页 → 点 **+ 新建配置**，给个名字（如"办公区"）。
2. 在规则表 **+ 添加规则**：
   - **目标 CIDR**：`168.168.0.0/16`（内网服务器段）
   - **接口**：下拉选你的内网网卡（如"以太网"）
   - **网关**：`auto`（自动取该网卡当前网关；也可填具体 IP）
   - **Metric**：`1`（小 = 优先）
   - **启用**：勾选
3. 再加一条 `172.16.0.0/16` → 同一内网网卡（如有终端段）。
4. **默认路由网卡** 下拉选 Wi-Fi（让其它流量走外网）。
5. **自动管理接口跃点数** 保持勾选（让默认路由确实走 Wi-Fi）。
6. 点 **保存**，再点 **设为活动**。

几秒内 **状态** 页会刷新：接口卡片、已下发路由表、最近一次 apply 结果。

> 任何时候网卡重连、插拔网线、DHCP 续约，GUI 进程都会在 ~2–5 秒内自动重新下发路由，无需手动干预。

## 4. 系统托盘

点窗口 ✕ → 隐藏到托盘（GUI 进程不退出，继续维护路由）。

- **左键单击**托盘图标 → 重新显示窗口
- **右键单击**托盘图标 → 菜单：
  - 显示主窗口
  - 立即重新应用路由
  - ─────
  - 退出 NetSwitcher（真正退出进程）
- **再双击 exe** → 不开第二个窗口，而是把已运行的窗口唤到最前（单实例锁）

## 5. 切换配置（Profile）

不同场景（办公 / 出差 / 家庭）可有不同规则集。在 **配置** 页左侧选另一个 profile → **设为活动**，路由即按新规则重下发。

## 6. 诊断

**诊断** 页：选 `ping` 或 `tracert`，输入目标，点运行。`tracert` 第一跳告诉你流量实际走了哪块网卡的网关。

**路由表** 页：完整 IPv4 路由表，按来源着色：
- 🟢 **本工具**：NetSwitcher 下发的
- 🟣 **疑似 VPN**：落在 VPN/虚拟网卡上
- ⚪ **系统**：其它
顶部搜索框可按目标/下一跳/接口过滤。

## 7. 状态页解读

- **接口卡片**：每块网卡一张，显示名称、是否连接、IPv4、网关、类型、Index。
- **已下发路由**：当前由本工具管理的路由。
- **跳过的规则**：写明哪条规则因什么原因没下发（接口名拼错、网卡没连、无网关）。下一轮网络变化自动重试。
- **冲突告警**：检测到 VPN 适配器在线或目标被外部覆盖时提示（**不会**自动覆盖 VPN 路由）。
- **底部**：最近一次 apply 的时间、原因。

## 8. 日志

**日志** 页实时滚动所有服务日志。下拉选级别（debug/info/warn/error），搜索框过滤。日志同时写入：

```
%ProgramData%\NetSwitcher\logs\netswitcher.log
```

按天滚动，单文件超过 50MB 也会滚动，保留 7 天。**设置**页可以运行时切换日志级别（立即生效 + 写回 config.json）。

## 9. 设置页

| 项 | 说明 |
|---|---|
| **开机自启** | 开关。开启 → 创建任务计划（登录时管理员启动，免 UAC）；关闭 → 删除 |
| **日志级别** | debug/info/warn/error，运行时切换 + 持久化 |
| **路由引擎** | "立即重新应用"按钮 + 引擎运行状态 |
| **日志文件** | "打开日志目录"按钮 |
| **关于** | 版本、权限、配置/状态文件路径 |

## 10. 不想用了

- **暂停**：托盘右键 → 退出 NetSwitcher（进程结束，路由不再被维护；已有的 runtime 路由保留到重启系统）。
- **取消开机自启**：设置页 → 关闭"开机自启"。
- **彻底删除**：退出程序 → 删除 exe + 删除 `%ProgramData%\NetSwitcher\`（配置和日志）。

## 11. 故障排查

| 现象 | 排查 |
|---|---|
| 顶栏显示"未提权" | 点弹窗"以管理员身份重启"，或设置页操作 |
| 路由没生效 | 状态页"已下发路由"里有没有？没有看"跳过的规则" |
| 内网还是不通 | 诊断页 `tracert 内网IP`，第一跳应是内网网卡网关 |
| 外网断了 | 默认路由网卡设对了吗？metric 管理开了吗？ |
| Wi-Fi 重连后断 | GUI 在跑吗？托盘有图标吗？日志里应有 `network_change` 的 apply 记录 |
| 配置保存报错 | 字段级错误会标红（CIDR 非法、网关非法、重复规则） |
| 与 VPN 冲突 | 状态页"冲突告警"会标出；本工具不主动覆盖 |
| 双击没反应 | 已有实例在跑，左键托盘图标唤出窗口 |
| 任务栏图标空白 | Windows 图标缓存——重启资源管理器或注销重登 |

## 12. 配置文件格式

`%ProgramData%\NetSwitcher\config.json`：

```json
{
  "version": 1,
  "activeProfile": "office",
  "profiles": [{
    "id": "office",
    "name": "办公区",
    "rules": [
      {"id": "r1", "destination": "168.168.0.0/16", "viaInterface": "以太网", "viaGateway": "auto", "metric": 1, "enabled": true}
    ],
    "defaultRouteInterface": "WLAN",
    "autoManageMetrics": true,
    "metricPolicy": {"preferredInterface": "WLAN", "preferredMetric": 10, "othersMetric": 50}
  }],
  "logLevel": "info"
}
```

GUI 监听该文件变化，外部编辑保存后自动 reload + 重新 apply。

## 13. 关于架构

NetSwitcher 是**单进程**架构：一个提权的 GUI 进程同时负责界面和路由维护。没有独立的 Windows 服务。

- **好处**：部署简单（单个 exe，无需安装服务），关窗口缩托盘后照常维护路由，开机自启靠任务计划。
- **代价**：只在你登录之后工作（登录界面 / 注销时不维护路由）。对个人台式机一般无所谓；如果需要"开机就维护、哪怕没人登录"，请参考技术方案文档里的服务模式（遗留，未在 GUI 中启用）。
